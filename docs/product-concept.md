# プロダクトコンセプト

## 1. 一言で

Mahjo（マージョ） ── 魅力的なキャラクターと一緒に麻雀の学習をしよう！ 役と点数の基礎が身につく個人用の学習Webアプリ。

ランダムな和了形を出題し、その手牌に対して「成立役の説明」と「点数の数え方」をビジュアルに見せる。

## 2. 目的（当面のゴール）

学習順序は 役 → 点数（まず成立役を理解し、その役を前提に符・点数を積む）。

1. 役を覚える練習（役モード）
2. 点数を数える練習（点数モード。符まで厳密に計算し、計算過程をビジュアルで段階表示）

## 3. 特徴（プロダクトの背骨）

Mahjo を Mahjo たらしめる、学び方と価値観。

| 特徴 | 内容 |
|---|---|
| 役 → 点数の順で学ぶ | 役が分からないと点数は数えられない。エンジンも UI もこの順で積む。 |
| ヒントは答えを言わない | 段階的に「ぼんやり→具体」へ小出し。最終段でも気づきを促す止まり。確定的な答えは回答後の解説（解説シーン）側に置く。 |
| プレッシャーをかけない | 学習継続を損なわないため、連続記録の強制や“サボると咎める”系はやらない。累計正答数はお祝い・アンロックに使うのみ。 |
| 正しさを絶対に妥協しない | 学習アプリで誤った点数を教えるのは致命的。役・符・翻・点数は常に正確に（担保のしかたは [testing.md](./dev/testing.md)）。 |

## 4. やらないこと

- 対局（実戦の打牌・進行）はしない。和了形に対する役・点数の学習に専念する。
- 対戦・ランキング・ソーシャルはやらない。競争の場ではなく、個人の学習のためのアプリ。

## 5. 競合と差別化

主目的は個人学習（市場で勝つことではない）が、立ち位置を明確にするため既存サービスを3層で整理する。

| 層 | 例 | 評価 |
|---|---|---|
| 静的な解説サイト・早見表 | [麻雀豆腐](https://majandofu.com/learn-calc) / [雀入門](https://www.jannyumon.jp/score/calculation/) など多数 | 読み物。Mahjo とは土俵が違い、競合とみなさない。 |
| 出題型クイズアプリ | [麻雀点数計算 超実践問題集](https://apps.apple.com/jp/app/id1493166793)（実戦12,175問）/ [麻雀点数計算マスター](https://play.google.com/store/apps/details?id=com.staminatech.mahjongstudy)（段階トレーニング）/ 海外 [Scoring Trainer](https://scoringtrainer.konbamwa.net/) 等 | **激戦区**。「ランダム和了形を出して当てさせる」だけだと量産型と正面衝突する。 |
| 役だけの暗記アプリ | [Mahjong Yaku Trainer](https://play.google.com/store/apps/details?id=com.keilinformatique.games.yaku) 等 | そこそこ存在。ただし役 → 点数を地続きで繋ぐものは少ない。 |

既存の出題型は「出題と正誤判定」までで止まる。**Mahjo が空いている所は、答えに辿り着く"過程"の支援**であり、以下3点の重なりが堀になる（§3 の背骨と対応）：

1. **答えを言わない段階ヒント** ── 既存は「不正解→正解と解説」で終わり。"ぼんやり→具体" に小出しして自力で気づかせる設計は見当たらない。
2. **クリックで内訳ハイライト連携** ── どの牌がこの符・この役か、を視覚で結ぶ（早見表アプリにはない）。
3. **キャラ（ペルソナ）× プレッシャーをかけない思想** ── 既存はストイックな問題集UIが主。学習継続を主目的に据えたキャラ駆動は空白地帯。

> 注意：「クイズ要素」単体を差別化の柱に置くと出題型クイズアプリに埋もれる。柱は **"気づきを促すヒント体験" と "キャラによる継続性"** に置き、クイズはその器と位置づける。
