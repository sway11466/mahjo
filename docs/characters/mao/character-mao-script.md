# まお セリフ script（全量・仮）

まおの全セリフの正本。**場面別セリフ（§1）＋着目ポイント別ヒントセリフ（§2）＋成立役の解説セリフ（§3）＋誤答の諭しセリフ（§4）**を網羅する。ヒント・解説は [hint-base.md](../../spec/hint-base.md) の役/ドラ キーをカバーする（突き合わせ＝バリデーション対象）。口調・性格はキャラ定義 [character-mao.md](./character-mao.md) §2 ペルソナに従う（です・ます基調／素直・優しい見習い）。§1 の必須場面（あいさつ／出題／正解／ミス／終了）は確定。ヒントセリフ（§2）と任意場面は仮で順次。§3 解説は通常役＋ドラを実作、役満・レア役は順次（parking lot）。

---

## 1. 場面別セリフ

場面ごとに表情（`Expression`）＋セリフ。複数からランダム表示（飽き対策）。場面の正準リスト・必須度は [session.md](../../spec/session.md) §4 場面（リアクション）の正準、セリフ側の規則は [character-guide.md](../character-guide.md) §2「セリフ」に従う。まおは neutral を持たず、あいさつ（greeting）も穏やかな笑み smile で迎える＝`reactions:{greeting:'smile'}` で上書き。他の場面は既定マップ通り。

### あいさつ（開始）（greeting → smile｜必須）
セッション開始の1回。
- こんにちは、今日も一緒にやってみましょう。
- よく来てくれましたね。さあ、はじめましょうか。
- 準備はいいですか？ いつでもどうぞ。

### 出題（各問）（dealing → thinking｜必須）
毎問の出題中（回答前）。答えは言わない。
- この手、じっくり見ていきましょう。
- 次は…どんな手でしょうね。
- さあ、この上がり。何が見えますか？
- 落ち着いて、ひとつずつ見ていきましょう。

### 正解時（correct → happy｜必須）
- 正解です！
- その通り、よく見抜きましたね。
- すごいです、わたしも嬉しい！
- ばっちりですね、いい調子です。
- さすがです。その目、たしかですね。

### ミス時（wrong → troubled｜必須）
感情寄りの短い寄り添い。方向づけ（どこを間違えたか）は §4 誤答の諭しが担い、誤答時は「ここ（寄り添い）＋ §4（諭し）」を結合して出す。
- 惜しい！
- あと少し、でしたね。
- だいじょうぶですよ。
- うーん、惜しい！

### 終了（結果）（finished → happy｜必須）
全8問を終えた結果画面。点数は画面側が出すので、お祝い・ねぎらいだけ。
- おつかれさまでした！ よくがんばりましたね。
- ぜんぶ見てきましたね。すてきな時間でした。
- 今日はここまで。また一緒にやりましょうね。

## 2. ヒントセリフ（着目ポイント別）

[hint-base.md](../../spec/hint-base.md) の各キーに、まおの言葉で段階セリフを当てる（**全キー網羅**＝バリデーション対象）。答え（役名・確定値）は言わない。配列 index = level（0=ぼんやり→具体）。現状の対象は41キー（役満15・`fu:base` は対象外。hint-base.md 冒頭）。

語り口：ヒントは使い魔（猫）がそっと示唆し、まおがひらめいて言葉にする体で書く（[decisions.md](../../decisions.md) 2026-06-10・A方向）。使い魔は声を持たないので、猫の所作は地の文に書かず視覚側（ヒントボタン＝使い魔）で表す。各段はまおの小さな“気づきのビート”（「…あ。」「…ん。」「ふふ。」等）から始め、答えは言わない原則は維持する。

### `generic`（汎用・idle ナッジ）
- …ん。まずは手牌ぜんたいを、ゆっくり眺めてみましょうか。

- あら、同じ牌が固まっていませんか？
- ねえ、あがり牌のまわりも見てみましょう。
- そうそう、場の状況も──場風（バカゼ）・自風（ジカゼ）・ドラ、忘れずに。

## 3. 解説セリフ（成立役の説明）

回答後の解説シーンで、まおが成立役を **1つずつ** 説明する文言（[screens.md](../../design/screens.md) §3 解説シーン）。ヒント（§2）と違い回答後なので **役名を出してよい**。1キー＝1役の説明（1〜2文）。該当牌などは画面側がハイライトするので、文は役の概念＋やさしい一言にとどめ、特定の牌列に依存させない（「ここ」程度の軽い指示はOK）。キーはヒントと同じ語彙（`yaku:<id>` ／ `dora`・`aka-dora`・`ura-dora`）。配列でなく1文（段は持たない）。

対象は通常役（[scoring-rules.md](../../spec/scoring-rules.md) §1.1）＋ドラ。役満（§1.2）・レア役（§1.3）は順次（parking lot で生成されるまで仮）。

### 役モード（`yaku:*`）

- `yaku:riichi`（リーチ）：リーチですね。門前（メンゼン）のまま聴牌（テンパイ）して宣言する――それだけで一翻つくんですよ。
- `yaku:double-riichi`（ダブルリーチ）：最初の巡目でのリーチ、「ダブルリーチ」です。早い決断、すてきでした。
- `yaku:ippatsu`（一発）：リーチのすぐあと、一巡以内であがれた「一発（イッパツ）」です。流れに乗れましたね。
- `yaku:menzen-tsumo`（門前清自摸和）：門前（メンゼン）のまま自分で引いてあがる「門前清自摸和（メンゼンツモ）」です。ツモのこの一枚ですね。
- `yaku:pinfu`（平和）：順子（ジュンツ）だけ・役のない雀頭（ジャントウ）・両面（リャンメン）待ち…符（フ）のつかない美しい形、「平和（ピンフ）」です。
- `yaku:tanyao`（断幺九）：端の1・9も字牌（ジハイ）も使わない「断幺九（タンヤオ）」です。2〜8だけ、すっきりしていますね。
- `yaku:iipeikou`（一盃口）：同じ並びの順子（ジュンツ）が二組、「一盃口（イーペーコー）」です。ここ、そっくりでしょう？
- `yaku:yakuhai-haku`（役牌 白）：三元牌（サンゲンパイ）の白をそろえた役牌（ヤクハイ）です。白は揃えるだけで一翻ですよ。
- `yaku:yakuhai-hatsu`（役牌 發）：發をそろえた役牌（ヤクハイ）です。緑のこの組ですね。
- `yaku:yakuhai-chun`（役牌 中）：中をそろえた役牌（ヤクハイ）です。赤いこの組です。
- `yaku:yakuhai-round`（場風）：場の風の牌をそろえた役牌（ヤクハイ）です。今の場と同じ風でしたね。
- `yaku:yakuhai-seat`（自風）：あなたの風の牌をそろえた役牌（ヤクハイ）です。自分の風と同じ、ここです。
- `yaku:sanshoku-doujun`（三色同順）：同じ並びを三つの色でそろえた「三色同順（サンショクドウジュン）」です。色がそろうと気持ちいいですね。
- `yaku:sanshoku-doukou`（三色同刻）：同じ数の刻子を三色、「三色同刻（サンショクドウコー）」です。なかなか珍しい形ですよ。
- `yaku:ittsuu`（一気通貫）：ひとつの色で1から9までまっすぐ一本、「一気通貫（イッキツウカン）」です。…わたしの好きな役なんです。
- `yaku:chanta`（混全帯幺九）：どの組にも端か字牌（ジハイ）が入る「混全帯幺九（チャンタ）」です。全部に么九（ヤオチュー）、見えますか。
- `yaku:junchan`（純全帯幺九）：端の1・9だけで字牌（ジハイ）を使わない「純全帯幺九（ジュンチャン）」です。チャンタより一段すごいんですよ。
- `yaku:chiitoitsu`（七対子）：面子（メンツ）を作らず、対子（トイツ）を七つそろえた「七対子（チートイツ）」です。ぜんぶ二枚ずつ、ですね。
- `yaku:toitoi`（対々和）：順子（ジュンツ）を使わず刻子（コーツ）だけでそろえた「対々和（トイトイ）」です。組ばかりの力強い形ですね。
- `yaku:sanankou`（三暗刻）：自分でそろえた暗刻（アンコー）が三つ、「三暗刻（サンアンコー）」です。ロンでない、ここの組たちですね。
- `yaku:sankantsu`（三槓子）：槓（カン）を三つ作った「三槓子（サンカンツ）」です。槓、大変だったでしょう。
- `yaku:honroutou`（混老頭）：1・9と字牌（ジハイ）だけの「混老頭（ホンロウトウ）」です。真ん中の牌がひとつも無いんですね。
- `yaku:shousangen`（小三元）：白・發・中のうち二つが刻子（コーツ）、一つが雀頭（ジャントウ）の「小三元（ショウサンゲン）」です。三元牌（サンゲンパイ）、集まりましたね。
- `yaku:honitsu`（混一色）：数牌（スーパイ）を一色だけ＋字牌（ジハイ）の「混一色（ホンイツ）」です。色がそろうと見ごたえありますね。
- `yaku:ryanpeikou`（二盃口）：一盃口（イーペーコー）が二組ぶん、「二盃口（リャンペーコー）」です。門前（メンゼン）でしか作れない形ですよ。
- `yaku:chinitsu`（清一色）：字牌（ジハイ）も混ぜず、一色だけでそろえた「清一色（チンイツ）」です。お見事です。
- `yaku:haitei`（海底摸月）：最後のツモであがる「海底摸月（ハイテイ）」です。ぎりぎりの一枚、引けましたね。
- `yaku:houtei`（河底撈魚）：場の最後の一枚でロンする「河底撈魚（ホウテイ）」です。最後まで見ていた成果ですね。
- `yaku:rinshan`（嶺上開花）：槓（カン）のあとの嶺上牌（リンシャンパイ）であがる「嶺上開花（リンシャンカイホー）」です。めずらしい巡り合わせですね。
- `yaku:chankan`（槍槓）：相手の加槓（カカン）に重ねてあがる「槍槓（チャンカン）」です。よく見ていましたね。

### 符（点数モード｜`fu:*`）

点数モードの解説で、符の発生源を **1つずつ** 説明する（[screens.md](../../design/screens.md) §3）。回答後なので符の数値を出してよい。各 `fu:*` キーはエンジンの符 item（[scoring-rules.md](../../spec/scoring-rules.md) §2・`src/engine/fu.ts`）に対応し、該当要素は画面側がハイライトする（副底・七対子＝手牌全体、ツモ＝ツモ表示、門前ロン＝門前ロン表示、待ち＝あがり牌、雀頭・刻子/槓子＝その牌）。`fu:base`・`fu:total` は符の段（ヒント）には出さない解説専用キー。

- `fu:base`（副底）：まずは副底（フーテイ）。どんな手も、ここから20符（フ）で始まります。土台ですね。
- `fu:menzen-ron`（門前ロン）：門前（メンゼン）のままロンであがると、10符（フ）。鳴かずに仕上げた、ごほうびですよ。
- `fu:tsumo`（ツモ）：自分で引いてあがると、2符（フ）。ツモの分ですね。
- `fu:wait`（待ち符）：受けの狭い待ち――単騎（タンキ）・嵌張（カンチャン）・辺張（ペンチャン）に付く2符（フ）です。両面（リャンメン）なら付きません。
- `fu:pair`（雀頭）：雀頭（ジャントウ）が役牌（ヤクハイ）だと、2符（フ）。場と自分の風が重なっていれば、もう少し多くなります。
- `fu:meld`（刻子・槓子）：刻子（コーツ）・槓子（カンツ）の符（フ）は、組み合わせで決まります。刻子は、明刻（ミンコー）が2符・暗刻（アンコー）が4符。端や字（么九〔ヤオチュー〕）なら、それぞれ倍の4符・8符です。槓子はさらに重く、明槓（ミンカン）8符・暗槓（アンカン）16符。么九の槓なら16符・32符にもなります。自分でそろえた組（暗）ほど、端の牌ほど、槓（カン）ほど、大きくなりますよ。（符の値は [scoring-rules.md](../../spec/scoring-rules.md) §2 面子符の表が正）
- `fu:kuipinfu`（喰いピンフ形）：鳴いていて符（フ）の付かない形は、20符のままにせず30符へ繰り上げます。
- `fu:chiitoi`（七対子の符）：七対子（チートイツ）は数えません。いつも25符（フ）と決まっています。
- `fu:total`（合計・切り上げ）：ぜんぶ足して、最後は10符（フ）の単位に切り上げます。これが、この手の符ですよ。

### ドラ

- `dora`（ドラ）：ドラです。ドラ表示の次の牌が、この手にありましたね。一枚ごとに一翻、お得ですよ。
- `aka-dora`（赤ドラ）：赤ドラです。赤い5が混じっていましたね。これも一枚で一翻。
- `ura-dora`（裏ドラ）：裏ドラです。リーチであがったご褒美、裏も乗りましたね。

## 4. 誤答の諭しセリフ（MistakeKind）

クイズで誤答したとき、選んだ誤答の `MistakeKind` に応じてまおがそっと諭す文言（[screens.md](../../design/screens.md) §3、中立の基準は [hint-base.md](../../spec/hint-base.md)「誤答の諭し素」）。全 `MistakeKind` を網羅（型 `Record<MistakeKind, string>` で強制）。**答え（正解値）は言わない**。誤答セリフは「§1 ミス時（寄り添い・rng）＋ ここ（具体の諭し）」を結合して出す（寄り添い＋具体）。種類が取れないときは §1 の寄り添いのみ。

- `dealer-swap`（親子取り違え）：もしかして、親と子を取り違えていませんか？ 自分の風が東かどうかで点数が変わりますよ。
- `dora-miss`（ドラ見落とし）：ドラ、見落としていませんか？ 赤い5や、リーチなら裏ドラも数えてみましょう。
- `tsumo-ron-swap`（ツモ/ロン取り違え）：ツモとロン、取り違えていませんか？ 符（フ）や払いが少し変わるんです。
- `fu-miscount`（符の数え違い）：符（フ）の数え、どこかで食い違ったかも。待ちや刻子（コーツ）、雀頭（ジャントウ）をていねいに見てみましょう。
- `han-miscount`（翻の数え違い）：翻（ハン）の数え、ひとつ抜けたかもしれませんね。役をもう一度、かぞえてみましょう。
