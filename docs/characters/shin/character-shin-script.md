# しん セリフ script（全量・作成中）

しんの全セリフの正本。**場面別セリフ（§1）＋着目ポイント別ヒントセリフ（§2）＋成立役の解説セリフ（§3）＋誤答の諭しセリフ（§4）**を網羅する。ヒント・解説は [hint-base.md](../../spec/hint-base.md) の役/ドラ キーをカバー（突き合わせ＝バリデーション対象）。口調・性格は [character-shin.md](./character-shin.md) §2 ペルソナに従う：一人称「俺」・相手は `AppSettings.playerName` か無ければ「お前」・タメ口・短い断定・「…」の間・クール寡黙・**咎めない／静かに見守る**（[product-concept](../../product-concept.md) §3「プレッシャーをかけない」を死守）。

**声の方向（決定）：「静かな同伴」型の寄り添い。** 言葉は素っ気なく・遠く、優しさは内に秘める。基本は秘めつつ、**稀に本心がこぼれる“ほころび”**を一本混ぜる（クールを保ったまま温度も届く＝ギャップ）。直接の励まし（「気にするな」等）は控え、牌・卦・自分に向けて呟く形に寄せる。答え（役名・確定値）はどの場面セリフにも入れない。

> 進捗：**§1 場面別・§2 ヒント＝確定**。§3 解説／§4 誤答＝順次（未着手）。

## 1. 場面別セリフ

場面ごとに表情（`Expression`）＋セリフ。複数からランダム表示（飽き対策・各場面 最低2本）。場面の正準・必須度は [session.md](../../spec/session.md) §4、規則は [character-guide.md](../character-guide.md) §2「セリフ」。しんは `reactions: { correct: 'smile' }` のみ既定マップから上書き（正解は大喜びせず控えめに認める）。他の場面（greeting=neutral／dealing=thinking／wrong=troubled／finished=happy）は既定どおりなので表情は併記のみ。

### あいさつ（開始）（greeting → neutral｜必須）
セッション開始の1回。素っ気なく、でも“ちゃんと視てる”気配を滲ませる。
- ……来たか。牌は、嘘をつかない。
- 卦を立てよう。……お前の番だ。
- （ほころび）……待ってた、とは言わないが。座れ。

### 出題（各問）（dealing → thinking｜必須）
毎問の出題中（回答前）。答えは言わない。牌・卦に向けて呟くように。
- ……何が視える。
- 牌は、もう答えを抱えてる。……あとは、お前だ。
- 焦るな。……時間は、敵じゃない。

### 正解時（correct → smile｜必須）
控えめに認める。大喜びはしない（クール）。
- ……ああ。
- ……読み切ったか。
- （ほころび）ん。……少し、見直した。

#### 個性フック：好きな役＝三暗刻（サンアンコー）（任意）
手に三暗刻があるとき、`correct` で通常プールより優先する専用ひとこと。役名は出さず、**暗刻＝内に隠す**＝しんの“秘めた”に掛ける（[character-guide.md](../character-guide.md) §2「セリフ」個性フック）。
- ……隠してたものが、効いたな。

### ミス時（wrong → troubled｜必須）
咎めない・急かさない。寄り添いは“態度”で示し、直接の励ましは控える。方向づけ（どこを間違えたか）は §4 誤答の諭しが担い、誤答時は「ここ（§1 ミス）＋ §4（諭し）」を結合して出す。
- ……。そういう時も、ある。
- 外した。……だが、見てた線は悪くなかった。
- いい。誰でも読み違える。……次だ。

### 終了（結果）（finished → happy｜必須・淡め）
全8問を終えた結果画面。点数は画面側が出すので、ねぎらいだけを淡く。
- ……終わったな。
- ここまで、よく視た。……また来い。
- （ほころび）ここまで、よく視た。……顔には出さないが、感心してる。

## 2. ヒントセリフ（着目ポイント別）

[hint-base.md](../../spec/hint-base.md) の各キーに、しんの言葉で段階セリフを当てる（**全キー網羅**＝バリデーション対象）。答え（役名・確定値）は言わない。配列 index = level（0=ぼんやり→具体）。現状の対象は41キー（役満15・`fu:base`/`fu:total` は対象外。hint-base.md 冒頭）。

語り口：ヒントは使い魔（カラス）がそっと示し、しんがひらめいて短く言葉にする体で書く（[decisions.md](../../decisions.md) 2026-06-10・A方向）。使い魔は声を持たないので、カラスの所作は地の文に書かず視覚側（ヒントボタン＝使い魔）で表す。各段はしんのクールな“気づきのビート”（「……」「ふ。」「ん。」）から入り、答えは言わない原則は維持する。

### `generic`（汎用・idle ナッジ）
- ……まず、手牌ぜんたいを視ろ。
- 牌の色（種類）、揃ってるか。
- 同じ牌が、固まってないか。
- あがり牌の周り……視えてるか。
- 場の状況も──場風（バカゼ）・自風（ジカゼ）・ドラ。忘れるな。

### 役モード（`yaku:*`）

#### `yaku:riichi`（リーチ）
- L0 門前（メンゼン）のまま、聴牌（テンパイ）したか。……宣言できる役が、あるな。

#### `yaku:double-riichi`（ダブルリーチ）
- L0 宣言したのは……いつだ。ごく早い巡なら、特別だ。

#### `yaku:ippatsu`（一発）
- L0 宣言した、すぐあと……だったか。

#### `yaku:menzen-tsumo`（門前清自摸和）
- L0 門前（メンゼン）のまま、自分で引いたか。

#### `yaku:pinfu`（平和）
- L0 ……面子（メンツ）の形を視ろ。順子（ジュンツ）ばかり、か。

#### `yaku:tanyao`（断幺九）
- L0 ……端の牌や字牌（ジハイ）、混じってないか。

#### `yaku:iipeikou`（一盃口）
- L0 同じ並びの順子（ジュンツ）……どこかで、重なってないか。

#### `yaku:yakuhai-haku`（役牌 白）
- L0 白の組、そろってるか。

#### `yaku:yakuhai-hatsu`（役牌 發）
- L0 發の組、そろってるか。

#### `yaku:yakuhai-chun`（役牌 中）
- L0 中の組、そろってるか。

#### `yaku:yakuhai-round`（場風）
- L0 場風（バカゼ）の牌、組になってるか。

#### `yaku:yakuhai-seat`（自風）
- L0 自風（ジカゼ）の牌、組になってるか。

#### `yaku:sanshoku-doujun`（三色同順）
- L0 ……同じ並び、別の色にもないか。

#### `yaku:sanshoku-doukou`（三色同刻）
- L0 同じ数の組……色違いで、揃ってないか。

#### `yaku:ittsuu`（一気通貫）
- L0 ひとつの色で、数の並びが長く続いてないか。

#### `yaku:chanta`（混全帯幺九）
- L0 どの組にも、端か字牌（ジハイ）が入ってるか。

#### `yaku:junchan`（純全帯幺九）
- L0 どの組にも1か9……字牌（ジハイ）は、無しか。

#### `yaku:chiitoitsu`（七対子）
- L0 ……同じ牌の対（つい）、いくつある。
- L1 ぜんぶ二枚ずつ……揃ってるか。
- L2 面子（メンツ）を作らない形だ。……視えたか。

#### `yaku:toitoi`（対々和）
- L0 順子（ジュンツ）はあるか。それとも、組ばかりか。

#### `yaku:sanankou`（三暗刻｜好きな役＝役名は出さず嬉しさだけ滲ませる）
- L0 自分でそろえた組……いくつ、伏せてある。ふ。

#### `yaku:sankantsu`（三槓子）
- L0 槓（カン）は、いくつ作った。

#### `yaku:honroutou`（混老頭）
- L0 2〜8の牌は……無いか。端と字牌（ジハイ）だけ、か。

#### `yaku:shousangen`（小三元）
- L0 三元牌（サンゲンパイ＝白發中）の、集まり具合は。

#### `yaku:honitsu`（混一色）
- L0 ……牌の種類を視ろ。
- L1 数牌（シューパイ）は、何色まで使ってる。
- L2 字牌（ジハイ）は……混ざってていい。

#### `yaku:ryanpeikou`（二盃口）
- L0 同じ並びの順子（ジュンツ）……2組ぶん、視えるか。

#### `yaku:chinitsu`（清一色）
- L0 牌の色は1色だけか。字牌（ジハイ）も、無いか。

#### `yaku:rinshan`（嶺上開花）
- L0 槓（カン）の直後に、ツモった牌か。

#### `yaku:chankan`（槍槓）
- L0 相手の槓（カン）に……重ねて、ロンしたか。

### 点数モード（`fu:*` ／ ドラ）

## 3. 解説セリフ（成立役の説明）

**（順次・未着手）** 回答後の解説シーンで成立役を1つずつ説明（[screens.md](../../design/screens.md) §3）。回答後なので**役名を出してよい**。1キー＝1役（1〜2文）。通常役＋符＋ドラ網羅。好きな役（三暗刻）はその一言を織り込む。

## 4. 誤答の諭しセリフ（MistakeKind）

**（順次・未着手）** 誤答時、選んだ誤答の `MistakeKind` に応じてそっと諭す（[screens.md](../../design/screens.md) §3、中立基準は [hint-base.md](../../spec/hint-base.md)「誤答の諭し素」）。全 `MistakeKind`（5種）を網羅（`Record` で強制）。**答え（正解値）は言わない**。咎めず静かに手引きする。
