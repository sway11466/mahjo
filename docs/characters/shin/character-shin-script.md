# しん セリフ script（全量・作成中）

しんの全セリフの正本。**場面別セリフ（§1）＋着目ポイント別ヒントセリフ（§2）＋成立役の解説セリフ（§3）＋誤答の諭しセリフ（§4）**を網羅する。ヒント・解説は [hint-base.md](../../spec/hint-base.md) の役/ドラ キーをカバー（突き合わせ＝バリデーション対象）。口調・性格は [character-shin.md](./character-shin.md) §2 ペルソナに従う：一人称「俺」・相手は `AppSettings.playerName` か無ければ「お前」・タメ口・短い断定・「…」の間・クール寡黙・**咎めない／静かに見守る**（[product-concept](../../product-concept.md) §3「プレッシャーをかけない」を死守）。

**声の方向（決定）：「静かな同伴」型の寄り添い。** 言葉は素っ気なく・遠く、優しさは内に秘める。基本は秘めつつ、**稀に本心がこぼれる“ほころび”**を一本混ぜる（クールを保ったまま温度も届く＝ギャップ）。直接の励まし（「気にするな」等）は控え、牌・卦・自分に向けて呟く形に寄せる。答え（役名・確定値）はどの場面セリフにも入れない。

> 進捗：**§1 場面別・§2 ヒント・§3 解説＝確定**。§4 誤答＝順次（未着手）。

## 1. 場面別セリフ

場面ごとに表情（`Expression`）＋セリフ。複数からランダム表示（飽き対策・各場面 最低2本）。場面の正準・必須度は [session.md](../../spec/session.md) §4、規則は [character-guide.md](../character-guide.md) §2「セリフ」。しんは `reactions: { correct: 'smile' }` のみ既定マップから上書き（正解は大喜びせず控えめに認める）。他の場面（greeting=neutral／dealing=thinking／wrong=troubled／finished=happy）は既定どおりなので表情は併記のみ。

### あいさつ（開始）（greeting → neutral｜必須）
セッション開始の1回。素っ気なく、でも“ちゃんと視てる”気配を滲ませる。
- ……来たか。牌は、嘘をつかない。
- 卦を立てよう。……お前の番だ。
- （ほころび）……待ってた、とは言わないが。座れ。

### 出題（各問）（dealing → thinking｜必須）
毎問の出題中（回答前）。答えは言わない。牌・卦に向けて呟くように。
- ……何が視える。
- 牌は、もう答えを抱えてる。……あとは、お前だ。
- 焦るな。……時間は、敵じゃない。

### 正解時（correct → smile｜必須）
控えめに認める。大喜びはしない（クール）。
- ……ああ。
- ……読み切ったか。
- （ほころび）ん。……少し、見直した。

#### 個性フック：好きな役＝三暗刻（サンアンコー）（任意）
手に三暗刻があるとき、`correct` で通常プールより優先する専用ひとこと。役名は出さず、**暗刻＝内に隠す**＝しんの“秘めた”に掛ける（[character-guide.md](../character-guide.md) §2「セリフ」個性フック）。
- ……隠してたものが、効いたな。

### ミス時（wrong → troubled｜必須）
咎めない・急かさない。寄り添いは“態度”で示し、直接の励ましは控える。方向づけ（どこを間違えたか）は §4 誤答の諭しが担い、誤答時は「ここ（§1 ミス）＋ §4（諭し）」を結合して出す。
- ……。そういう時も、ある。
- 外した。……だが、見てた線は悪くなかった。
- いい。誰でも読み違える。……次だ。

### 終了（結果）（finished → happy｜必須・淡め）
全8問を終えた結果画面。点数は画面側が出すので、ねぎらいだけを淡く。
- ……終わったな。
- ここまで、よく視た。……また来い。
- （ほころび）ここまで、よく視た。……顔には出さないが、感心してる。

## 3. 解説セリフ（成立役の説明）

回答後の解説シーンで、しんが成立役を **1つずつ** 説明する（[screens.md](../../design/screens.md) §3 解説シーン）。ヒント（§2）と違い回答後なので **役名を出してよい**。1キー＝1役の説明（1〜2文）。該当牌などは画面側がハイライトするので、文は役の概念＋短い一言にとどめ、特定の牌列に依存させない（「ここ」程度の軽い指示はOK）。キーはヒントと同じ語彙（`yaku:<id>` ／ `fu:*` ／ `dora`・`aka-dora`・`ura-dora`）。配列でなく1文（段は持たない）。声はしんのクール・断定（「俺」「…」の間）で、咎めず静かに認める。好きな役（三暗刻）はその一言を織り込む。

対象は通常役（[scoring-rules.md](../../spec/scoring-rules.md) §1.1）＋符＋ドラ。役満（§1.2）・レア役（§1.3）は順次（parking lot で生成されるまで仮）。

### 役モード（`yaku:*`）

- `yaku:riichi`（リーチ）：門前（メンゼン）のまま聴牌（テンパイ）して、宣言した。……それがリーチだ。一翻、付く。
- `yaku:double-riichi`（ダブルリーチ）：最初の巡で、もう宣言してた。ダブルリーチだ。……早い決断だな。
- `yaku:ippatsu`（一発）：リーチの、すぐあと。一巡以内であがった……一発（イッパツ）だ。流れに乗ったな。
- `yaku:menzen-tsumo`（門前清自摸和）：鳴かず、自分で引いた。門前清自摸和（メンゼンツモ）……この一枚だ。
- `yaku:pinfu`（平和）：順子（ジュンツ）だけ、雀頭（ジャントウ）は役にならない牌、待ちは両面（リャンメン）。符（フ）の付かない形……平和（ピンフ）だ。きれいに収まってる。
- `yaku:tanyao`（断幺九）：端も字牌（ジハイ）も、使ってない。2〜8だけ……断幺九（タンヤオ）だ。
- `yaku:iipeikou`（一盃口）：同じ並びの順子（ジュンツ）が、二組。一盃口（イーペーコー）だ。……重なってる、ここだ。
- `yaku:yakuhai-haku`（役牌 白）：三元牌（サンゲンパイ）の白を、そろえた。役牌（ヤクハイ）だ。揃えるだけで、一翻。
- `yaku:yakuhai-hatsu`（役牌 發）：發をそろえた、役牌（ヤクハイ）だ。緑の、この組。
- `yaku:yakuhai-chun`（役牌 中）：中をそろえた、役牌（ヤクハイ）だ。赤の、この組。
- `yaku:yakuhai-round`（場風）：場風（バカゼ）の牌を、そろえた。役牌（ヤクハイ）だ。今の場と、同じ風。
- `yaku:yakuhai-seat`（自風）：自風（ジカゼ）の牌を、そろえた。役牌（ヤクハイ）だ。お前の風と、同じ。
- `yaku:sanshoku-doujun`（三色同順）：同じ並びを、三つの色で。三色同順（サンショクドウジュン）だ。……揃うと、気分がいい。
- `yaku:sanshoku-doukou`（三色同刻）：同じ数の刻子（コーツ）を、三色。三色同刻（サンショクドウコー）だ。……珍しい形だ。
- `yaku:ittsuu`（一気通貫）：ひとつの色で、1から9まで一本。一気通貫（イッキツウカン）だ。まっすぐ、通ってる。
- `yaku:chanta`（混全帯幺九）：どの組にも、端か字牌（ジハイ）が入ってる。混全帯幺九（チャンタ）だ。全部に么九（ヤオチュー）……視えるか。
- `yaku:junchan`（純全帯幺九）：端の1・9だけ、字牌（ジハイ）は無し。純全帯幺九（ジュンチャン）だ。チャンタより、一段上だ。
- `yaku:chiitoitsu`（七対子）：面子（メンツ）を作らず、対子（トイツ）を七つ。七対子（チートイツ）だ。ぜんぶ、二枚ずつ。
- `yaku:toitoi`（対々和）：順子（ジュンツ）を使わず、刻子（コーツ）だけ。対々和（トイトイ）だ。……組ばかりの、力強い形。
- `yaku:sanankou`（三暗刻｜好きな役）：自分で伏せた暗刻（アンコー）が、三つ。三暗刻（サンアンコー）だ。表に出さず仕込む……俺の、好きな役だ。
- `yaku:sankantsu`（三槓子）：槓（カン）を、三つ作った。三槓子（サンカンツ）だ。……よく、ここまで。
- `yaku:honroutou`（混老頭）：2〜8が、ひとつも無い。端と字牌（ジハイ）だけ……混老頭（ホンロウトウ）だ。
- `yaku:shousangen`（小三元）：白・發・中のうち、二つが刻子（コーツ）、一つが雀頭（ジャントウ）。小三元（ショウサンゲン）だ。三元牌（サンゲンパイ）、集まったな。
- `yaku:honitsu`（混一色）：数牌（シューパイ）は一色、あとは字牌（ジハイ）だけ。混一色（ホンイツ）だ。……色が、揃ってる。
- `yaku:ryanpeikou`（二盃口）：一盃口（イーペーコー）が、二組ぶん。二盃口（リャンペーコー）だ。門前（メンゼン）でしか、作れない。
- `yaku:chinitsu`（清一色）：字牌（ジハイ）も混ぜず、一色だけ。清一色（チンイツ）だ。……見事だ。
- `yaku:haitei`（海底摸月）：最後のツモで、あがった。海底摸月（ハイテイ）だ。ぎりぎりの一枚、引いたな。
- `yaku:houtei`（河底撈魚）：場の、最後の一枚でロン。河底撈魚（ホウテイ）だ。最後まで、視てたな。
- `yaku:rinshan`（嶺上開花）：槓（カン）の、あとの嶺上牌（リンシャンパイ）。嶺上開花（リンシャンカイホー）だ。……珍しい巡り合わせだ。
- `yaku:chankan`（槍槓）：相手の加槓（カカン）に、重ねた。槍槓（チャンカン）だ。よく、見てた。

### 符（点数モード｜`fu:*`）

点数モードの解説で、符の発生源を **1つずつ** 説明する（[screens.md](../../design/screens.md) §3）。回答後なので符の数値を出してよい。各 `fu:*` キーはエンジンの符 item（[scoring-rules.md](../../spec/scoring-rules.md) §2・`src/engine/fu.ts`）に対応し、該当要素は画面側がハイライトする。`fu:base`・`fu:total` は符の段（ヒント）には出さない解説専用キー。

- `fu:base`（副底）：まず副底（フーテイ）。どんな手も、20符（フ）から始まる。……土台だ。
- `fu:menzen-ron`（門前ロン）：門前（メンゼン）のまま、ロンであがった。10符（フ）……鳴かずに仕上げた分だ。
- `fu:tsumo`（ツモ）：自分で引いた。2符（フ）……ツモの分だ。
- `fu:wait`（待ち符）：単騎（タンキ）・嵌張（カンチャン）・辺張（ペンチャン）は、2符（フ）。両面（リャンメン）・双碰（シャンポン）には、付かない。
- `fu:pair`（雀頭）：雀頭（ジャントウ）が役牌（ヤクハイ）なら、2符（フ）。場と自分の風が重なれば……もう少し。
- `fu:meld`（刻子・槓子）：刻子（コーツ）と槓子（カンツ）の符（フ）は、組み合わせで決まる。伏せた組ほど、端の牌ほど、槓（カン）ほど……重くなる。（値は [scoring-rules.md](../../spec/scoring-rules.md) §2 面子符の表が正）
- `fu:kuipinfu`（喰いピンフ形）：鳴いていて、符（フ）の付かない形。20符のままにせず……30符へ繰り上げる。
- `fu:chiitoi`（七対子の符）：七対子（チートイツ）は、数えない。いつも25符（フ）と、決まってる。
- `fu:total`（合計・切り上げ）：ぜんぶ足して、最後は10符（フ）の単位に切り上げる。……これが、この手の符だ。

### ドラ

- `dora`（ドラ）：ドラ表示の、次の牌……持ってたな。一枚で、一翻だ。
- `aka-dora`（赤ドラ）：赤い5が、混じってた。赤ドラだ。これも一枚で、一翻。
- `ura-dora`（裏ドラ）：リーチであがった……裏も、乗ったな。裏ドラだ。

### 役満・レア役（順次）
`yaku:kokushi` ほか役満（[scoring-rules.md](../../spec/scoring-rules.md) §1.2）・レア役（§1.3）は、生成される parking lot で実作（仮・未）。

## 4. 誤答の諭しセリフ（MistakeKind）

**（順次・未着手）** 誤答時、選んだ誤答の `MistakeKind` に応じてそっと諭す（[screens.md](../../design/screens.md) §3、中立基準は [hint-base.md](../../spec/hint-base.md)「誤答の諭し素」）。全 `MistakeKind`（5種）を網羅（`Record` で強制）。**答え（正解値）は言わない**。咎めず静かに手引きする。
