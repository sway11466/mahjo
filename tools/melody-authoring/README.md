# 主旋律づくりツール（melody-authoring）

キャラクターごとの BGM「主旋律」を、耳で確かめながら作るためのオーサリングツール。

BGM はコード生成方式（音源ファイルを持たず Web Audio で合成）を採用予定で、二層構成 ── 主旋律（キャラごとの決まったメロディ）＋即興音（合間を飾る毎回ちがう音）── で組む方針。本ツールはそのうち **主旋律を1本ずつ書き起こす** ための道具。

> BGM の実現方式・度数記法・即興音のセグメント配列の正は [docs/design/sound.md](../../docs/design/sound.md)「BGM の実現方式」（判断の記録は [ADR-0003](../../docs/adr/ADR-0003-bgm-code-generation.md)）。本ツールは主旋律を書き起こすための道具で、パラメータ表示（`music = { ... }`）はオーサリング用の見せ方。本番 BGM 機能（`src/`）実装時に共有ロジックを抽出する（下記「位置づけ」）。

## 使い方

ビルド不要。`index.html` をブラウザで開くだけ（バニラ HTML/JS・依存なし）。

画面は3ブロック：

1. パラメーター（UI）── 楽器・主音（キー）・オクターブ・モード（調）・テンポ・明るさ・余韻。
2. パラメーター（テキスト・編集可）── 上の設定＋メロディをテキストで表示。**直接編集でき、UI と双方向に同期**する（`//` 以降のコメントは表示専用で、パース時は無視）。
3. 再生 ── ピアノロールと再生ボタン（ループ／ベースの下支えを切替可）。

開いた時点で中立の初期メロディが入っている。テキストや UI を書き換えながら旋律を詰め、確定した各キャラの主旋律は profile（`docs/characters/<id>/character-<id>.md`）へ書き出す。

## 度数記法（メロディの書き方）

- 音符：度数 `1`〜`5`（そのキャラの音階を低い方から。主音＝`1`）。半音を含まない5音なので `1`〜`5` だけで書ける。
- オクターブ：`'` で1つ上、`,` で1つ下（重ねると複数）。例 `1'`＝主音の1オクターブ上、`5,`＝5度の1オクターブ下。
- 長さ：`:拍数` を付ける（省略で1拍）。例 `3:2`＝度数3を2拍。
- 休符：`0`（`0:2` で2拍の休み）。
- 区切り：空白。

例：`1 2 4:2 3 2 1:2 5, 1 2:2 3 2 1:2`

モード（調）を切り替えると、同じ度数でも実際の音程が変わる（`宮`＝明るい長音階ペンタ／`羽`＝翳りのある短音階ペンタ など5種）。旋律の形はそのままに、明暗の色だけを変えられる。

## 音の合成について

鳴っている音はすべて合成で、録音サンプルは使っていない。撥弦（琴）は Karplus-Strong、鐘はFM合成、木琴・笛・弓はオシレーター＋エンベロープ。素の Web Audio API のみ（Tone.js 等のライブラリ不使用）。

## 位置づけ

- 開発・オーサリング用のツール。アプリ本体（`src/`）にも公開サイト（`public/`）にも同梱しない（`tools/` 配下の独立物）。
- 記法パーサと音の合成ロジックは、いずれ実装する本番 BGM 機能（`src/`）と重複する。現時点では無理に共通化せず本ツール内に閉じて持ち、本番機能を作るときに共有ロジックを抽出する。
